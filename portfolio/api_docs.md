# API & Database Schema

## Core API Endpoints
- `POST /api/v1/auth/login` - Authenticate user and return JWT.
- `GET /api/v1/cameras` - List all registered cameras.
- `GET /api/v1/alerts` - Fetch recent security alerts.
- `GET /api/v1/health` - Get system hardware and DB status.
- `GET /api/v1/analytics` - Fetch aggregated crime stats for dashboard charting.

## Database Schema (PostgreSQL via Supabase)

### Table: `users` (Managed by Supabase Auth)
- `id`: UUID
- `email`: String
- `role`: String (admin, operator, viewer)

### Table: `cameras`
- `id`: UUID
- `name`: String
- `location`: String
- `rtsp_url`: String
- `is_active`: Boolean

### Table: `alerts`
- `id`: UUID
- `camera_id`: UUID (FK)
- `alert_type`: String
- `severity`: String (critical, high, medium)
- `snapshot_url`: String
- `created_at`: Timestamp

### Table: `criminals`
- `id`: UUID
- `first_name`: String
- `last_name`: String
- `face_encoding`: Vector/Array (Generated by InsightFace)
